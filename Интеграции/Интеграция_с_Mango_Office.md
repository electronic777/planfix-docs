## Сохранение данных динамического коллтрекинга


При использовании [динамического коллтрекинга](https://www.mango-office.ru/products/calltracking/) возможно сохранение его данных в аналитику звонков в ПланФиксе 


Для этого измените аналитику Звонок (и Пропущенный звонок, если необходимо) и добавьте в неё следующие поля типа Строка (можно только необходимые из них): 


  * uid - идентификатор пользователя (уникальная кука из браузера)

  * widget_id - идентификатор виджета динамического коллтрекинга, к которому относится пользователь

  * widget_name - наименование виджета динамического коллтрекинга, к которому относится пользователь

  * product_id - идентификатор продукта, к которому относится виджет

  * ga_cid - идентификатор пользователя в Google Analytics

  * ya_cid - идентификатор пользователя в Яндекс Метрике

  * rs_cid - идентификатор пользователя в Roistat

  * ip - IP адрес пользователя

  * region_id - идентификатор региона из Биллинга, который присвоен пользователю (будут одинаковыми, если в настройках коллтрекинга не включена мультирегиональность)

  * duration - время в секундах с момента захода пользователя на сайт

  * device - устройство посетителя: mobile или desktop

  * custom - дополнительный параметр от клиента (ограничение по длине 100 символов)


информация о местоположении пользователя: 


  * location_country_code - ISO код страны

  * location_region_code - код региона

  * location_region - название региона o city - название города


объект, источник/канал, по которому пользователь пришёл на сайт: 


  * channel_source - utm source

  * channel_medium - utm medium

  * channel_campaign - utm campaign

  * channel_content - utm content

  * channel_term - utm term


текущая страница, на которой находится пользователь: 


  * current_page_url - абсолютный адрес страницы, например <http://example.ru/orders/123?param=1>

  * current_page_date - дата и время открытия страницы в формате UTC (по ISO)

  * current_page_title - заголовок страницы
