Письма внешним контактам могут отправляться двумя способами: 


  * С личного (корпоративного) адреса электронной почты пользователя.

  * С общего для нескольких (или всех) пользователей адреса электронной почты компании.


ПланФикс предполагает использование либо первого, либо второго варианта. Если данные учетной записи почты настроены и на уровне компании, и на уровне конкретного сотрудника, то сработают настройки на уровне сотрудника. Это дает возможность в одном аккаунте для части сотрудников использовать общий почтовый ящик (настроив его на уровне компании), а для другой части – личные адреса e-mail (настроив их в личных карточках сотрудников). 


  * Обратите внимание: в почтовом сервере должна быть включена возможность подключения по протоколу POP3.


  

Настройка внешних почтовых адресов сотрудников используется, если письма внешним контактам должны отправляться с личного (в том числе личного корпоративного) адреса пользователя. 


Это наиболее распространенный вариант, так как он соответствует привычному для большинства компаний порядку, когда каждый сотрудник общается с внешними контактами от своего имени. Если вам нужно, чтобы несколько (или все) сотрудники общались с одного общего корпоративного адреса электронной почты, необходимо использовать [ настройку внешних адресов компании](Контакты/Настройка_внешних_почтовых_адресов_компании.md). 


  * Настройка личного адреса сотрудника производится во вкладке **Внешние адреса** [ карточки сотрудника](Интерфейс/Страница_пользователя.md) или в разделе **Управление аккаунтом** — **Работа с e-mail —** Исходящие сообщения**—** Внешние адреса сотрудников**.**


  * Выбрав нужного сотрудника, вы попадаете в список адресов, с которых он может отправлять свои сообщения из ПланФикса. По умолчанию, в списке пусто. Тут же есть возможность добавить внешний адрес по кнопке.


  * Клик на кнопку **Добавить внешний адрес** вызывает окно настроек учетной записи электронной почты. В нем есть возможность выбрать настройки по умолчанию для одной из популярных почтовых служб или ввести данные другого (в том числе корпоративного) почтового сервера.


## Обратите внимание


При активированной настройке **Получать ответы на этот же адрес** письма-ответы от клиента будут приходить на ваш адрес электронной почты и оттуда они посредством переадресации попадут в ПланФикс: 


![EFgJef.png](images/EFgJef.png)


  

Помимо технических данных, необходимых для организации отправки email-сообщений через внешний почтовый сервер, в этом окне вы можете выбрать дополнительные опции. В том числе указать проекты, для которых будет использоваться этот адрес: 


![rumhfi.jpg](images/rumhfi.jpg)


  

Настройка **По умолчанию** означает, что этот внешний адрес будет автоматически выбираться для указанных проектов тем пользователям, у которых есть к нему доступ. 


Такой подход позволяет вести в одном аккаунте ПланФикса работу от лица нескольких компаний/брендов и выбирать, с какого адреса будут отправляться письма клиентам в рамках конкретного проекта. 


## Важно


  * Обязательно протестируйте внесенные настройки, нажав на кнопку **Протестировать учетную запись**. Если тест не проходит, сохранять эти изменения нельзя, иначе ПланФикс не сможет отправлять сообщения вашим контрагентам.


  * Если в используемом вами почтовом сервисе включена двухфакторная авторизация, может понадобиться пройти [ дополнительную процедуру подключения](Коммуникации/Активация_внешнего_почтового_адреса_при_использовании_двухфакторной_авторизации.md)


## Ошибка при отправке письма клиенту без доступа в Планфикс


Если при включении отправки почты через корпоративный адрес вы получаете сообщение об ошибке, прочтите внимательно текст ошибки. Как правило, ее возвращает ваш корпоративный сервер и текст ошибки содержит причину, по которой письмо не было отправлено или доставлено. Самые частые среди них мы привели на [ отдельной странице](Контакты/Ошибка_при_отправке_письма_клиенту.md).
