Рассмотрим идеологию использования пользовательских полей на примере добавления в систему сущности "Заявка клиента". Согласно [идеологии системы](https://planfix.ru/ideologiya/), заявка клиента — это задача, обладающая дополнительными свойствами. Для того, чтобы использовать всю мощь инструментария задач ПланФикса для работы с заявками, настраиваем для них отдельный Объект: 


  * Создаем новый [Объект](Прочее/Объекты.md) под названием "Заявка клиента".

  * Добавляем в него нужные [поля](Прочее/Создание_пользовательского_поля.md), характеризующие заявку (тип и размеры заказываемого изделия, вид заказанной услуги, адрес доставки и т.п., в зависимости от деятельности компании).

  * [Скрываем поля](Прочее/Карточка_объекта.md), которые не нужны для работы с заявками.

  * Сохраняем получившийся Объект и получаем возможность работать в ПланФиксе с задачами нового типа.

  * Создавая задачи созданного нами типа, сотрудники получают возможность заполнять данными нужные поля заявки.

  * В дальнейшей работе задачи-заявки могут отбираться при помощи [фильтров](Задачи/Фильтры_задач.md), выводиться в [отчетах](Аналитика/Отчеты.md) и списках [Планировщика](Прочее/Планировщик.md).

  * Отбор удобно проводить по полю, специфичному именно для заявок — так обычные задачи, созданные по стандартному шаблону, не будут мешаться с заявками.


  

Этот подход позволяет одновременно работать в одном аккаунте ПланФикса с разными по сути бизнес-процессами, используя общий инструментарий системы и разделяя задачи по отдельным веткам. 


## Сквозное использование


Созданное пользовательское поле правильно использовать во всех Объектах ПланФикса, для которых оно актуально. Например, пользовательское поле "Номер транспортного средства" можно использовать и в Объекте "Заявка клиента", и в Объекте "Закупка", и в Объекте "Технические работы". Это позволит при необходимости вести по этому полю сквозную статистику. Например, выводить в отчетах общее использование транспорта с группировкой по типам использования, проектам и т.п.
