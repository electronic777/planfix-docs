С момента настройки [ связи аккаунтов](Связь_аккаунтов.md "Связь аккаунтов") все уведомления о новых задачах и действиях для пользователя будут отправляться в его основной аккаунт. При этом действует следующая логика: 


  * Если в аккаунте еще нет такой задачи, она будет автоматически создана. В нее попадут все комментарии и файлы, которые были доступны пользователю на этот момент.

  * Если задача уже была создана, новые действия будут попадать в нее автоматически (при уведомлении пользователя о них).

  * Если задаче (в любом аккаунте) будет выставлен другой [ стандартный статус](Какой_смысл_в_статусах_задач_.md "Какой смысл в статусах задач?") \- он изменится во всех связанных аккаунтах.

  * [ Иерархия подзадач](Подзадачи.md "Подзадачи") в связанных аккаунтах передаются, при условии, что и надзадача и ее подзадача перешли по связи в связанный аккаунт.

  * В системе реализован автоматический подбор для новой задачи по связи [ проекта](Проекты.md "Проекты") с именем, точно совпадающим с именем проекта из аккаунта, откуда она пришла. Т.е. если в основном аккаунте есть проект, с названием, точно совпадающим с названием проекта, из того аккаунта, откуда задача пришла - задача поместится в него автоматически. Если такого проекта нет, задача будет создана без проекта.

  * Когда задача поступает по связи из другого аккаунта, для каждого пользователя, добавленного к ней в качестве исполнителя или участника, автоматически создается карточка контакта с признаком установленной связи. Это позволяет выбирать их в блоке "Уведомить об этом" при добавлении нового комментария в ходе работы над задачей. Они получат эти уведомления в своем основном аккаунте и смогут продолжить работу над задачей.

  * Чек-листы при связи аккаунтов синхронизируются, если постановщик или исполнители в задаче (где добавлен чек-лист) из другого аккаунта.


## Ограничения передачи данных


  * По связи передаются только изменения задачи сделанные сотрудниками, находящимися в ней в роли постановщика, участника или исполнителя.

  * На текущий момент между аккаунтами не передается следующая информация: 

    * [Пользовательские статусы задачи](Пользовательские_статусы_задачи.md "Пользовательские статусы задачи")

    * [Пользовательские поля задачи](Пользовательские_поля_задачи.md "Пользовательские поля задачи")

    * [ Аналитики](Аналитика.md "Аналитика"), добавленные в задачи

    * [Напоминания](Напоминания.md "Напоминания")

    * [ Аудиторы задачи](Аудитор_задачи.md "Аудитор задачи") и [ аудиторы проекта](Аудитор_проекта.md "Аудитор проекта")
