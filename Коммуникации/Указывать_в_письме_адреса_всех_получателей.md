ПланФикс позволяет работать с внешними по отношению к системе контактами в режиме, максимально приближенном к обычной работе по e-mail: 


  * Все участники общения получают общее письмо, в котором перечислены все его получатели.

  * При ответе в почтовой программе каждый из них использует вариант **Ответить всем** , чтобы его ответ получили остальные участники общения.


Признак **Указывать в письме адреса всех получателей** наряду с признаком [Не отправлять контактам уведомления о комментариях, добавленных другими участниками по e-mail](Контакты/Не_отправлять_контактам_уведомления_о_комментариях_добавленных_другими_участниками_по_e-mail.md) помогает настроить ПланФикс на такой сценарий использования почты. Активация этого признака приводит к тому, что уведомления о задачах и действиях, как минимум одним из получателей которых является внешний контакт, отправляются одним, общим для всех получателей письмом. При этом в поле **Кому** этого письма перечисляются email-адреса всех получателей - и внешних контактов, и сотрудников. 


Активируется признак по следующему пути: 


    **Управление аккаунтом** — **Работа с помощью e-mail** — **Уведомление внешних контактов** — **Указывать в письме адреса всех получателей**


  

При использовании этой настройки необходимо учитывать следующее: 


  * Письмо-уведомление будет отправлено по соответствующему [ шаблону для внешних контактов](Контакты/Шаблоны_писем-уведомлений_для_контактов_без_доступа_в_ПланФикс.md). Соответственно, если среди получателей письма есть сотрудники, то они получат письмо по шаблону для контакта, т.к. все адресаты получают одно общее письмо.

  * Письмо-уведомление будет отправлено на внешние адреса сотрудников, которые указаны в их профиле и используются для уведомлений на email. Соответственно, эти же адреса будут перечислены в списке получателей письма-уведомления и именно их увидит клиент.

  * Если у сотрудника отключены уведомления на email, в письме будет указан адрес, установленный при помощи дополнительной настройки **Адреса сотрудников, отключивших уведомления на email** : 

    * **Не включаются в список получателей** \- сотрудник не получит уведомления и его адреса не будет в списке получателей.

    * **Указывается внутренний email-адрес** \- в списке получателей будет указан [ внутренний email-адрес](Задачи/Как_поставить_задачу_по_e-mail.md) уведомленного сотрудника.

    * **Указывается внешний email-адрес** \- в списке получателей будет указан email-адрес из карточки уведомленного сотрудника. При этом он получит письмо, вне зависимости от того, что у него [ выключены](Коммуникации/Подписки_и_уведомления.md) уведомления по e-mail для сообщений данного типа.

  * Письмо-уведомление будет отправлено с адреса, который настроен в системе для отправки писем внешним контактам: [ внутреннего адреса сотрудника](Задачи/Как_поставить_задачу_по_e-mail.md) (автора сообщения), либо [ настроенного](Коммуникации/Отправка_писем_с_внешнего_почтового_ящика.md) внешнего e-mail адреса.

  * Если активирована [ отправка истории переписки](Контакты/История_переписки_в_письмах_для_контактов_без_доступа_в_ПланФикс.md) и среди получателей есть несколько контактов, в письмо будет включена история только для одного из них.

  * Если среди получателей уведомления нет ни одного контакта, будет использована стандартная система уведомлений сотрудников. В том числе будут отправлены стандартные системные письма тем сотрудникам, у которых [ включены](Коммуникации/Подписки_и_уведомления.md) уведомления по e-mail.
