Правила настраиваются в разделе [ Управление аккаунтом](Общие/Аккаунт.md) — Работа с e-mail — Правила обработки для ответов в задачи. 


Эти правила распространяются на: 


  * [ Ответы на письма-уведомления из задачи](Автоматизация/Как_добавить_действие_с_помощью_e-mail_.md) (о создании задачи, о новом действии и т.п.)

  * Письма, отправленные напрямую на [ e-mail адрес задачи](Задачи/Как_добавить_в_задачу_внешнее_письмо_.md)


Каждое входящее сообщение последовательно проверяется на соответствие условиям срабатывания каждого из настроенных правил. В случае выполнения условий правила, проводятся описанные в правиле операции, влияющие на создаваемое по этому сообщению действие. 


## Пример правила для обработки писем, приходящих на адрес задачи


Это правило проверяет, нет ли во входящем письме текста «С уважением» и если находит его, то при формировании текста комментария удаляет из оригинального текста письма все, начиная с этой метки (при этом сам оригинальный текст письма, который доступен для просмотра по специальной ссылке в добавленном действии, остается без изменений).

Аналогичные правила могут быть добавлены для других общеупотребительных оборотов (например, Best regards и т.п.), с которых начинаются подписи. Этот же подход позволит избавиться от неинформативных автоматических текстов в поступающих вам письмах от других систем. Все это позволяет уменьшить длину добавленного в ПланФикс текста комментария и улучшить его читабельность.
