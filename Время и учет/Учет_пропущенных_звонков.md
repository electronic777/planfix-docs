Для учета звонков, при подключении интеграции с [ виртуальными АТС](Прочее/Виртуальные_АТС.md) в аккаунт добавляются две системные [ аналитики](Аналитика/Аналитика.md): [Звонок](Прочее/Звонок.md) и **Пропущенный звонок**. 


Если входящий звонок был пропущен и используемая АТС умеет отдавать информацию об этом, ПланФикс автоматически добавит в [ карточку звонящего контакта](Контакты/Карточка_контакта.md) новое действие с аналитикой **Пропущенный звонок** : 


  * Если у [ контакта](Контакты/Контакты.md) есть [ответственный](Прочее/Ответственный.md), он будет уведомлен об этом действии.

  * Наличие аналитики позволяет легко строить отчеты по пропущенным звонкам в нужном разрезе.

  * Также вы можете использовать стандартный отчет [Пропущенные звонки](Прочее/Пропущенные_звонки.md) для просмотра этой информации.


## Дополнительно


  * Существует возможность добавлять к аналитике поля типа [Дата](Прочее/Дата.md) и [Период времени](Время и учет/Период_времени.md) для построения по этим аналитикам [ Планировщика](Прочее/Планировщик.md).
