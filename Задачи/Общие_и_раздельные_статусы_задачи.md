[ Задача](Задачи.md) может в один и тот же момент времени находиться для разных исполнителей в разных статусах. Это позволяет пользователю отрабатывать задачи вне зависимости от действий коллег и видеть состояние задачи, актуальное именно для него. 


  


Статусы задач   

---  

Раздельные | Общие   

Новая | Завершенная   

В работе* | В работе   

Отклонённая | Отложенная   

Выполненная* | Выполненная   

| Отменённая   

  

  


  * _Статус «В работе» трактуется как раздельный, когда используется вместе со статусом «Новая» и механизмом принятия задачи_.

  * _Статус «Выполненная» трактуется как раздельный, когда в задаче есть чек-лист с несколькими пунктами, назначенными на разных исполнителей. Если один из исполнителей отметит свои пункты выполненными и переведет задачу в статус «Выполненная», то этот статус будет актуален только для него, а для остальных и для постановщика задача будет находиться «В работе»_.


  

Для примера представим задачу с двумя исполнителями: 


  * Один исполнитель принял задачу и она для него находится в статусе **В работе**. Его коллега пока не принял задачу, она для него находится в статусе **Новая**. Это позволяет первому сотруднику видеть задачу в списке задач в работе, а второму - в списке новых задач. При этом постановщик видит задачу в статусе **Новая** до тех пор, пока ее не приняли все исполнители. Статусы **Новая** и **В работе** \- **раздельные** (еще может использоваться термин **Частичный статус**).

  * При этом достаточно одному исполнителю перевести задачу в статус **Завершенная** , чтобы для всех других исполнителей и для постановщика она тоже стала завершенной. Статус "Завершенная" - **общий**. Задача находится в таком статусе одновременно для всех пользователей, вне зависимости от их роли в ней.

  * Статус **Выполненная** в общем случае является общим, то есть достаточно одному сотруднику перевести задачу в **Выполненная** и она станет таковой для всех исполнителей и постановщика. Но есть ситуация, когда этот статус может стать раздельным: это происходит автоматически, если в задаче есть [ чек-лист](../Общие/Чек-лист.md) с несколькими пунктами, назначенными на разных исполнителей. Если один из исполнителей отметит свои пункты выполненными и переведет задачу в статус **Выполненная** , то этот статус будет актуален только для него, а для остальных и для постановщика она будет находиться **В работе**. Таким образом, при переводе задачи в статус Выполненная проверяется, есть ли не отмеченные пункты чек-листа, за которые ответственны другие исполнители. И если они есть, статус меняется только для этого сотрудника, если нет - для всей задачи. Только после того, как все исполнители отметят свои пункты чек-листа и переведут задачу в Выполненная, постановщик увидит задачу в этом статусе.


  

Для каждого [ пользовательского статуса](Пользовательские_статусы_задачи.md) есть возможность указывать, является он общим или раздельным. 


  

На вкладке **Детали задачи** можно увидеть, в каком статусе она находится для каждого из исполнителей - имя исполнителя отображается цветом, соответствующим цвету статуса. По клику на имя пользователя можно изменить индивидуальный статус для каждого исполнителя, если у вас есть на это [ права](Таблица_прав_доступа_к_задаче_по_ролям.md).
