Для [ полей](../Поля и формы/Пользовательские_поля.md) задачи, контакта и проекта есть возможность установить признак **Обязательно для заполнения**. Настройка обязательности заполнения производится в режиме редактирования шаблона соответствующего объекта: 


![azlnu4.png](images/azlnu4.png)


  

Находим нужное поле и открываем его настройки кликом на троеточие и активируем признак **Обязательно для заполнения** : 


![MfH0BG.png](images/MfH0BG.png)


  

В режиме создания/редактирования задачи (проекта, контакта) поле с активированным признаком обязательности заполнения будет отмечено красной звездочкой. Сохранить задачу, не заполнив такое поле, не получится. 


## Обязательное заполнение системных полей


  * С апреля 2019 года для системных полей проектов, задач и контактов также доступна настройка обязательности заполнения.

  * Обязательность заполнения системных полей настраивается так же, как и в случае с пользовательскими полями.


## Важно


  * Настройка обязательности заполнения поля производится на уровне шаблона задачи / контакта, которые определяют тип объекта системы. Вследствие этого, одно и то же поле может быть обязательным для заполнения в одном типе объектов и необязательным в другом. Например, поле «Адрес» может быть необязательным для заполнения в задаче-запросе клиента, но обязательным к заполнению в задаче-наряде на доставку заказа клиенту.
