**Шаблон проекта** позволяет быстро создавать [проекты](Проекты/Проекты.md) с заранее определенными параметрами: принадлежность к группе проектов, роли в проекте, шаблоны задач внутри проекта и так далее. 


Шаблон проекта дает возможность также создать структуру стандартных задач, из которых обычно состоит проект данного типа. 


По умолчанию, в каждом аккаунте есть один шаблон проекта под названием **Стандартный шаблон**. По этому шаблону создаются проекты, если не выбран другой шаблон. Этот шаблон можно отредактировать, но нельзя удалить. 


## Создание шаблона проекта


  * Для создания нового шаблона перейдите в раздел **Проекты** — на панели слева **Шаблоны проектов** — нажмите кнопку **Новый шаблон проекта**.

  * По клику открывается форма создания нового шаблона проекта, в которой можно задать:


  1. **Набор реквизитов проекта** — какие поля будут присутствовать в создаваемом проекте.

  2. **Данные, предзаписанные в поля проекта** — например, группу проектов, надпроект, роли в проекте и т.п. данные, чтобы сэкономить время на их заполнение в процессе создания проектов.


  * После нажатия **Сохранить и перейти** , происходит переход к списку [задач этого шаблона проекта](Задачи/Шаблоны_задач_в_рамках_шаблонов_проектов.md). В этом списке отображаются не сами задачи, а [шаблоны](Задачи/Шаблоны_задач.md), по которым они будут созданы. По умолчанию, список пуст.


## Использование шаблона проекта


  * При создании проекта выберите нужный шаблон из списка.

  * В форму создания проекта подгрузятся все заранее установленные в шаблоне параметры.


## Связь проекта и шаблона


Проект, созданный по шаблону, будет связан с ним во всем, что касается формы, но не содержания шаблона. То есть: 


  * если в шаблон будут внесены изменения по форме (например, добавлено новое поле), то эти изменения автоматически отобразятся во всех проектах, созданных по этому шаблону;

  * если в шаблон будут внесены изменения по содержанию (например, изменена группа проектов), то это никак не отразится на созданных по данному шаблону проектах.

  * если в шаблоне проекта будут добавлены или удалены шаблоны задач, то это не приведет к созданию задач по этим шаблонам в проектах, созданных ранее по измененному шаблону проекта.

  * Права доступа может менять автор шаблона, а также его руководители и администратор аккаунта.


## Дополнительная информация


  * В шаблоне проекта вы можете задать [ шаблон задачи, по которому будут создаваться задачи в проекте](Задачи/Шаблоны_задач_в_проекте_по_умолчанию.md). Эта настройка будет унаследована всеми проектами, созданными по этому шаблону.

  * В шаблон проекта можно добавить [повторяющиеся задачи](Задачи/Повторяющиеся_задачи.md). Они будут создаваться с заданной периодичностью после создания проекта по такому шаблону. Если необходимо, чтобы повторяющаяся задача создавалась независимо от создания проекта по шаблону, укажите для таких задач отдельный шаблон повторения, не привязанный к шаблону проекта.

  * В шаблоне проекта вы можете выбрать [ вид отображения задач проекта](Задачи/Вид_отображения_задач_проекта.md). Эта настройка будет унаследована всеми проектами, созданными по этому шаблону.


## Обратите внимание


  * Количество шаблонов проектов, которые вы можете создать, соответствует ресурсам выбранного [пакета](Прочее/Какой_пакет_выбрать_.md).

  * В течение пробного периода ограничений на создание шаблонов проектов нет.
