Дубли [ контактов](Контакты.md) — это одна из распространенных проблем в CRM, которая выражается в присутствии в списке контактов с несколькими карточками и пересекающейся информацией, которые соответствуют одному человеку или компании. 


Для защиты от дублирования контактов в ПланФиксе используется несколько механизмов: 


  * Уникальность по электронной почте

  * Проверка уникальности по имени

  * Проверка уникальности по номеру телефона


## Уникальность по электронной почте


ПланФикс не разрешит добавить контакт или компанию с электронной почтой, которая уже используется в аккаунте. При этом учитываются как email-адреса контактов и компаний, так и электронная почта сотрудников. При попытке добавить контакт с адресом электронной почты, который уже используется в ПланФиксе, система выдаст соответствующее сообщение:

## Проверка уникальности по имени


При вводе имени и фамилии контакта, ПланФикс в фоновом режиме ищет совпадения вводимых данных с уже добавленными в систему контактами. Если совпадение найдено, появится указатель со ссылкой на него: 


![qJ4tg7.png](images/qJ4tg7.png)


  

Кликнув на ссылку, можно просмотреть данные найденного контакта и решить, нужно ли добавлять новый. 


Проверка совпадения по имени, отчеству и фамилии работает следующим образом: 


  * Если при создании нового контакта вводится имя при пустых полях "Отчество" и "Фамилия", то ищется совпадение по части слова.


    **Пример** : в системе есть контакт Максим Иванов, вы добавляете новый контакт и набираете в поле "Имя" Макс, система находит совпадение и информирует вас об этом.


  * Если вы вводите значение в поле "Отчество" или "Фамилия" и при этом остальные два поля ФИО не заполнены, то система ищет полное совпадение.


    **Пример** : в системе есть контакт Наталья Петрова, вы при создании нового контакта указываете фамилию Петров, система не будет считать это совпадением;


  * Если информация вводится сразу в несколько полей ФИО, дубликат ищется сразу по всем заполненным полям. При этом поиск по имени выполняется по части слова, а поиск по отчеству и фамилии учитывает только полные совпадения.


    **Пример** : в системе есть контакт Максим Иванов, вы добавляете контакт Макс Иванов, система находит совпадение и информирует вас об этом.

    **Пример** : в системе есть контакт Наталья Петрова, вы добавляете контакт Татьяна Петрова, система не будет считать это совпадением;

    **Пример** : в системе есть контакт Наталья Петрова, вы добавляете контакт Наталья Сергеевна Петрова, система отметит его как возможный дубль и выдаст сообщение об этом.


## Проверка уникальности по номеру телефона


Как и в случае с именем, при вводе номера телефона, ПланФикс в фоновом режиме ищет совпадения вводимых данных с уже добавленными в систему контактами. Если совпадение найдено, появится указатель со ссылкой на него: 


![Z9DikW.png](images/Z9DikW.png)


  

Кликнув на ссылку, можно просмотреть данные найденного контакта и решить, нужно ли добавлять новый. 


## Важно


  * ПланФикс не разрешит вам добавить контакт с уже существующим в системе адресом электронной почты, т.к. этот адрес может использоваться для переписки с контактом и/или его доступа в систему. Исключение: [импорт контактов](Импорт_контактов.md) с опцией "Считать все контакты новыми", при которой наличие совпадений в существующей базе контактов не проверяется.

  * Во всех других случаях — при совпадении имени, телефона и т.п. — система предупредит вас о возможных дублях, но оставит решение за вами.

  * Если в ПланФиксе оказались дубли контактов, вы можете [объединить их в один контакт](Объединение_контактов.md). При этом данные по этим контактам, накопленные в системе, также будут объединены.
