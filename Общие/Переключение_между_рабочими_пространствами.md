По умолчанию, у каждого сотрудника есть доступ только к [ базовому рабочему пространству](Базовое_рабочее_пространство.md), которое содержит полный функционал ПланФикса. Если помимо базового, сотруднику [ дан доступ](../Доступ и права/Доступ_к_рабочему_пространству.md) к другим [ рабочим пространствам](Рабочие_пространства.md), у него в главном меню отображается пункт переключения между рабочими пространствами: 


![xSEDjf.png](images/xSEDjf.png)


  

Для переключения в нужное рабочее пространство необходимо кликнуть на значок ракеты в главном меню и выбрать нужное рабочее пространство. 


**Примечания:**


  * Если у сотрудника есть доступ только к базовому рабочему пространству, пункт меню рабочих пространств ("ракета") для него отображаться не будет

  * Администраторы системы всегда видят пункт меню рабочих пространств


  


## Переключение между рабочими пространствами для клиентов


По умолчанию, каждый [ клиент](../Контакты/Внешний_пользователь.md) имеет доступ к базовому рабочему пространству, которое содержит часть функционала ПланФикса, [ доступную для внешних пользователей](../Доступ и права/Ограничения_по_работе_в_ПланФиксе_для_внешних_пользователей.md). Также ему может быть [ дан доступ](../Доступ и права/Доступ_к_рабочему_пространству.md) к дополнительным рабочим пространствам. При этом логика переключения между рабочими пространствами для клиента несколько отличается: 


  * Если помимо базового, у клиента есть доступ к одному дополнительному рабочему пространству, он автоматически будет работать в этом дополнительном рабочем пространстве. Меню переключения между рабочими пространствами он не увидит и переключиться в базовое рабочее пространство не сможет. Это сделано для того, чтобы не выводить лишний пункт ("ракета") в главном меню.

  * Если у клиента есть доступ к двум и более рабочим пространствам помимо базового, он видит меню рабочих пространств и может переключаться между ними, включая базовое рабочее пространство.


  


## Важно


  * Возможность переключиться в то или иное рабочее пространство не приводит к тому, что пользователь увидит в интерфейсе ПланФикса какие-то данные, не предназначенные для него. Доступ к данным полностью определяется [ правами доступа](../Доступ и права/Права_доступа_в_ПланФиксе.md) для конкретного пользователя.
