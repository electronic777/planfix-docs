Как правило, **аудитор проекта** \- это один из руководителей компании, который осуществляет наблюдение за ходом проекта. 


Аудитор: 


  * видит все задачи, файлы и действия проекта (за исключением [скрытых комментариев](Скрытые__(приватные_.md)_%D0%BA%D0%BE%D0%BC%D0%BC%D0%B5%D0%BD%D1%82%D0%B0%D1%80%D0%B8%D0%B8 "Скрытые \(приватные\) комментарии"));

  * может редактировать все задачи проекта;

  * может удалять задачи проекта и восстанавливать их из Корзины, но не может очищать Корзину (т.е. проводить окончательное удаление задач из нее);

  * может самостоятельно добавлять новые задачи любому исполнителю проекта, а также комментарии в любой задаче проекта

  * может редактировать и удалять проект.


По этой причине представителей клиента лучше не подключать к проекту в качестве аудиторов - это дает им слишком широкие полномочия. 


## Важно


  * Аудитор проекта задается при редактировании карточки проекта.

  * У проекта может быть несколько аудиторов.
