Для учета звонков, при подключении интеграции с [ виртуальными АТС](Виртуальные_АТС.md "Виртуальные АТС") в аккаунт добавляются две системные [ аналитики](Аналитика.md "Аналитика"): [Звонок](Звонок.md "Звонок") и **Пропущенный звонок**. 


Если входящий звонок был пропущен и используемая АТС умеет отдавать информацию об этом, ПланФикс автоматически добавит в [ карточку звонящего контакта](Карточка_контакта.md "Карточка контакта") новое действие с аналитикой **Пропущенный звонок** : 


  * Если у [ контакта](Контакты.md "Контакты") есть [ответственный](Ответственный.md "Ответственный"), он будет уведомлен об этом действии.

  * Наличие аналитики позволяет легко строить отчеты по пропущенным звонкам в нужном разрезе.

  * Также вы можете использовать стандартный отчет [Пропущенные звонки](Пропущенные_звонки.md "Пропущенные звонки") для просмотра этой информации.


## Дополнительно


  * Существует возможность добавлять к аналитике поля типа [Дата](Дата.md "Дата") и [Период времени](Период_времени.md "Период времени") для построения по этим аналитикам [ Планировщика](Планировщик.md "Планировщик").
