**Клиент-менеджер проекта** отвечает за контакт с клиентом, для которого осуществляется проект. Если к проекту [ подключен представитель клиента](Внешний_пользователь.md "Внешний пользователь"), то он может ставить задачи только клиент-менеджеру своего проекта, остальные сотрудники компании ему недоступны (он не видит их в списке при создании новой задачи). 


Клиент-менеджер может добавить к комментарию файл или аналитику, а также уведомить о комментарии добавленного участника. Ему доступны только те задачи в проекте, в которых он участвует лично. 


Клиент-менеджер проекта задается при создании или редактировании карточки проекта. 


Если в шаблоне проекта клиент-менеджер не указан, то при создании проекта по шаблону поле "Клиент-менеджер" заполнится автоматически создателем проекта. Это нужно для того, чтобы не попадать в ситуации, когда клиенту некому ставить задачи (напомним, представитель клиента может ставить задачи только клиент-менеджеру). 


У проекта может быть несколько клиент-менеджеров. 


## Важно


  * Клиент-менеджер не имеет доступа ко всем задачам проекта - он видит только те задачи, в которых непосредственно участвует.

  * Если у сотрудника есть необходимость видеть задачи по проекту, можно назначить его [участником проекта](Участники_по_умолчанию.md "Участники по умолчанию").

  * Если у сотрудника есть необходимость управлять задачами проекта (редактировать, завершать, удалять и т.п.), его можно назначить [аудитором проекта](Аудитор_проекта.md "Аудитор проекта").

  * Клиент-менеджер при отсутствии других аудиторов в проекте при создании задачи добавится и в исполнители, и в аудиторы. Это сделано для того, чтобы сотрудник мог что-то делать с задачей поставленной клиентом. Если в проекте указан другой аудитор, то клиент-менеджер аудитором задачи не станет.

  * Роль клиент-менеджера не даёт ему прав аудитора.

  * У Клиент-менеджера проекта есть доступ (только просмотр) ко всем файлам, загруженным в проект.
