Если вы не хотите давать [доступ клиенту](Внешний_пользователь.md "Внешний пользователь") в свой аккаунт ПланФикса, или клиент не горит желанием использовать новую систему, тогда работу с ним можно организовать при помощи электронной почты: 


  * создайте новый контакт в разделе [Контакты](Контакты.md "Контакты");

  * укажите имя клиента и его e-mail;

  * в разделе **Права доступа к контакту** активируйте чек-бокс **Отображать в списке выбора участников задачи**.


Это даст возможность добавить клиента в нужные задачи в качестве исполнителя или участника, при этом доступ в ПланФикс клиент не получит. 


**Важно:** если клиент написал вам [ письмо, которое стало задачей в ПланФиксе](Как_поставить_задачу_по_e-mail.md "Как поставить задачу по e-mail"), его карточка контакта создается автоматически, вы можете пропустить этот шаг и просто общаться с ним по задаче. 


## Работа по задачам


Если необходимо согласовать с клиентом какой-то вопрос или сообщить ему о результатах: 


  * добавьте новый комментарий к задаче, к которой подключен этот клиент;

  * напишите текст комментария, адресованный клиенту;

  * добавьте (при необходимости) файлы, с которыми хотите его ознакомить;

  * выделите его в блоке **Контакты** ;

  * сохраните комментарий.


Клиент получит письмо с вашей информацией на e-mail, который вы добавили ему как контакту в ПланФиксе. Ответ клиента на это письмо [ будет добавлен в задачу](Как_добавить_действие_с_помощью_e-mail_.md "Как добавить действие с помощью e-mail?") новым комментарием. 


## Какую информацию получает клиент


  * В общем случае, клиенту уходят сообщения только о тех действиях, о которых его уведомляют. Поэтому вы можете управлять видимостью для клиента действий, производимых по задаче, в том числе свободно переписываться внутри задачи с коллегами, не уведомляя об этом клиента - сообщений об этом он не получит.


  * Будьте внимательны, отвечая на сообщения от ПланФикса по email: ваш ответ будет отправлен всем получателям письма, на которое вы отвечаете, поэтому если одним из адресатов был клиент, то он также получит ваш ответ. При помощи параметра [Не отправлять контактам уведомления о комментариях, добавленных другими участниками по e-mail](Не_отправлять_контактам_уведомления_о_комментариях_добавленных_другими_участниками_по_e-mail.md "Не отправлять контактам уведомления о комментариях добавленных другими участниками по e-mail") можно ограничить получение клиентами таких ответов.


  * Системные уведомления (о переводе статуса задачи, подключении/отключении исполнителей и т.п.) клиенту НЕ отправляются.


  * Клиенту, [ поставившему задачу по почте](Как_поставить_задачу_по_e-mail_.md "Как поставить задачу по e-mail?") на внутренний адрес сотрудника компании, НЕ отправляется автоматическое сообщение о ее принятии либо просрочке


## Дополнительно


  * Клиентам, с которыми вы переписываетесь из ПланФикса, автоматически и бесплатно предоставляется [ Личный кабинет](Личный_кабинет.md "Личный кабинет"). В нём клиенты без расширенного доступа в систему могут видеть переписку по всем вопросам.


## Важно


  * Мы не рекомендуем использовать одинаковый e-mail для нескольких контактов, так как при отправке сообщения на этот e-mail система не сможет определить, какому из контактов отправить сообщение и отправит первому найденному. Для каждого контакта целесообразно указывать отдельный e-mail, не совпадающий с другими.
