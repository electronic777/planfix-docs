В дополнительных параметрах шаблона задачи можно выбрать вариант подключения исполнителей: 


  * **У задачи одновременно несколько исполнителей**. Вариант по умолчанию, при котором исполнителями задачи являются все пользователи, добавленные в поле **Исполнитель**.


  * **Подзадача для каждого исполнителя**. Для каждого исполнителя, указанного в поле "Исполнитель" или входящего в указанную в этом поле группу, будет создана отдельная подзадача к этой задаче, в которой он станет единственным исполнителем ([Подробности](Отдельная_задача_каждому_исполнителю.md)).


  * **Пункт чек-листа для каждого исполнителя**. Для каждого исполнителя, указанного в поле "Исполнитель" или входящего в указанную в этом поле группу, к этой задаче будет добавлен пункт чек-листа, в котором он будет указан в качестве исполнителя. Отметить такой пункт чек-листа выполненным может только этот исполнитель, другим он будет недоступен. Этот инструмент позволяет проводить ознакомление сотрудников с инструкциями или другой важной информацией ([пример](https://www.youtube.com/watch?v=yQ4V6nLTyJQ)).


  * **Исполнители устанавливаются по очереди**. Позволяет распределить задачи равномерно между несколькими исполнителями. Эту опцию удобно применять в работе отдела продаж или службы поддержки. Обратите внимание, если в исполнителях перечислен список пользователей, ПланФикс первым выберет любого из них, независимо от того, в каком порядке они расположены в списке.


  * **Развернуть группы на индивидуальных исполнителей**. Опция позволяет развернуть группу на поимённый список исполнителей. Обычно используется в опросах или при ознакомлении сотрудников с рабочими инструкциями.


  * [ **Первый принявший - исполнитель, остальные - участники**](Конкурентный_режим_подключения_исполнителей.md). Задает определенную логику поведения при принятии задачи, очевидную из названия самого варианта. Используется для организации конкурентного приема задач при работе отдела продаж, службы поддержки и т.п. Суть этого подхода в том, что тот, кто первым принимает новую задачу (заявку, лид и т.п.), тот и становится ответственным за нее.


  * [ **Первый принявший - исполнитель, остальные - отключаются**](Конкурентный_режим_подключения_исполнителей.md). Та же область применения, что и у предыдущего варианта, но немного отличающаяся логика работы. Если в предыдущем варианте другие исполнители остаются в задаче в роли "Участники" и тем самым имеют к ней доступ, то в этом варианте они этот доступ утрачивают.


## Важно


  * В любом из вариантов подключения используются только те пользователи, которые были включены в поле **Исполнитель**.

  * Полный перечень вариантов подключения исполнителей доступен только на этапе создания задачи.
