У каждого [ проекта](Проекты/Проекты.md) в ПланФиксе есть собственный виртуальный адрес электронной почты, который имеет следующий вид: 


    **project+123@аккаунт.planfix.ru**


    где:


    


  * **123** это номер проекта;

  * **аккаунт** \- имя вашего аккаунта ПланФикса.


  

Адрес сгененирован автоматически, возможности изменить его нет. Увидеть email-адрес проекта можно на панели [Детали проекта](Проекты/Детали_проекта.md) — в поле **Email проекта**. 


Письмо, отправленное на этот адрес, будет преобразовано в задачу в этом проекте. При этом: 


  * Если в проекте указан [ шаблон задачи по умолчанию](Задачи/Шаблоны_задач_в_проекте_по_умолчанию.md), задача будет создана на основании этого шаблона.

  * Если в проекте указаны [исполнители по умолчанию](Задачи/Исполнители_по_умолчанию.md) или [участники по умолчанию](Прочее/Участники_по_умолчанию.md), они будут добавлены в создаваемую задачу.


## Правила обработки входящих сообщений, приходящих на адреса проектов


В разделе [ Управление аккаунтом](Общие/Аккаунт.md) (доступен администраторам аккаунта) — **Работа с e-mail** — **Правила обработки для проектов** могут быть настроены правила обработки входящих сообщений, приходящих на адреса проектов: 


  * Каждое письмо, приходящее на адрес проекта, последовательно проверяется на соответствие условиям срабатывания каждого из настроенных правил.


  * В случае выполнения условий правила, проводятся описанные в правиле операции, влияющие на создаваемую по этому сообщению задачу.


  * Возможности правил аналогичны [ правилам создания задачи](Задачи/Правила_создания_задач_по_e-mail.md) по письмам, приходящим на [ внутренние адреса сотрудников](Задачи/Как_поставить_задачу_по_e-mail.md).
