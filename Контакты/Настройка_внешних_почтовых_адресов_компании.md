Письма внешним контактам могут отправляться двумя способами: 


  * С личного (корпоративного) адреса электронной почты пользователя.

  * С общего для нескольких (или всех) пользователей адреса электронной почты компании.


ПланФикс предполагает использование либо первого, либо второго варианта. Если данные учетной записи почты настроены и на уровне компании, и на уровне конкретного сотрудника, то сработают настройки на уровне сотрудника. Это дает возможность в одном аккаунте для части сотрудников использовать общий почтовый ящик (настроив его на уровне компании), а для другой части – личные адреса e-mail (настроив их в личных карточках сотрудников). 


  * Обратите внимание: в почтовом сервере должна быть включена возможность подключения по протоколу POP3.


  

Настройка внешних почтовых адресов компании используется, если письма внешним контактам должны отправляться с одного, общего для нескольких (или всех) сотрудников адреса электронной почты компании. Как правило, такой подход используется для организации работы службы поддержки, где операторы обезличены и общаются с клиентами от лица своей организации. 


Если вам нужно, чтобы каждый сотрудник общался с внешними контактами со своего персонального (в том числе персонального корпоративного) адреса электронной почты, необходимо использовать [ настройку внешних адресов сотрудников](Настройка_внешних_почтовых_адресов_сотрудников.md). 


  * Настройка внешних адресов компании производится в разделе **Управление аккаунтом** — **Работа с e-mail** — **Исходящие сообщения** — **Внешние адреса компании**.


  * Выбрав этот пункт, вы попадаете в список адресов, с которых компания может отправлять свои сообщения из ПланФикса. По умолчанию, в списке пусто. Тут же есть возможность добавить внешний адрес по кнопке.


  * Клик на кнопку **Добавить внешний адрес** вызывает окно настроек учетной записи электронной почты. В нем есть возможность выбрать настройки по умолчанию для одной из популярных почтовых служб или ввести данные другого (в том числе корпоративного) почтового сервера.


Помимо технических данных, необходимых для организации отправки email-сообщений через внешний почтовый сервер, в этом окне вы можете выбрать дополнительные опции. 


В частности, вы можете ограничить доступ к созданному адресу, предоставив его только выбранным сотрудникам и группам: 


![WsVmQ1.png](images/WsVmQ1.png)


  


## Важно


  * Обязательно протестируйте внесенные настройки, нажав на кнопку **Протестировать учетную запись**. Если тест не проходит, сохранять эти изменения нельзя, иначе ПланФикс не сможет отправлять сообщения вашим контрагентам.

  * Если в используемом вами почтовом сервисе включена двухфакторная авторизация, может понадобиться пройти [ дополнительную процедуру подключения](../Коммуникации/Активация_внешнего_почтового_адреса_при_использовании_двухфакторной_авторизации.md)


## Использование внешних email-адресов компании


В настройках e-mail адреса компании вы можете указать проекты, для которых будет использоваться этот адрес: 


![uUwzwF.png](images/uUwzwF.png)


  

Такой подход позволяет вести в одном аккаунте ПланФикса работу от лица нескольких компаний/брендов и выбирать, с какого адреса будут отправляться письма клиентам в рамках конкретного проекта. В других проектах, а также в задачах без проекта такой адрес будет недоступен для выбора, чтобы снять возможность отправки письма с него в результате случайной ошибки пользователя при выборе из списка адресов. 


Если не указывать проекты, для которых доступен этот адрес, он будет доступен во всех проектах. Выбрать адрес, с которого будет осуществляться отправка письма внешнему контакту, можно при добавлении действий (для этого включите современный вид отображения интерфейса ПланФикса): 


![Vl8qsM.png](images/Vl8qsM.png)


  

**Совет:** Добавьте как минимум один адрес компании без привязки к конкретному проекту для того, чтобы письма внешним контактам из ПланФикса уходили с этого адреса в случаях, когда работа идет в проектах, у которых не установлен "свой" адрес, или в задачах без проекта. 


## Ошибка при отправке письма клиенту без доступа в Планфикс


Если при включении отправки почты через корпоративный адрес вы получаете сообщение об ошибке, прочтите внимательно текст ошибки. Как правило, ее возвращает ваш корпоративный сервер и текст ошибки содержит причину, по которой письмо не было отправлено или доставлено. Самые частые среди них мы привели на [ отдельной странице](Ошибка_при_отправке_письма_клиенту.md).
