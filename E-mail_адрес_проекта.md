У каждого [ проекта](Проекты.md "Проекты") в ПланФиксе есть собственный виртуальный адрес электронной почты, который имеет следующий вид: 


    **project+123@аккаунт.planfix.ru**


    где:


    


  * **123** это номер проекта;

  * **аккаунт** \- имя вашего аккаунта ПланФикса.


  

Адрес сгененирован автоматически, возможности изменить его нет. Увидеть email-адрес проекта можно на панели [Детали проекта](Детали_проекта.md "Детали проекта") — в поле **Email проекта**. 


Письмо, отправленное на этот адрес, будет преобразовано в задачу в этом проекте. При этом: 


  * Если в проекте указан [ шаблон задачи по умолчанию](Шаблоны_задач_в_проекте_по_умолчанию.md "Шаблоны задач в проекте по умолчанию"), задача будет создана на основании этого шаблона.

  * Если в проекте указаны [исполнители по умолчанию](Исполнители_по_умолчанию.md "Исполнители по умолчанию") или [участники по умолчанию](Участники_по_умолчанию.md "Участники по умолчанию"), они будут добавлены в создаваемую задачу.


## Правила обработки входящих сообщений, приходящих на адреса проектов


В разделе [ Управление аккаунтом](Аккаунт.md "Аккаунт") (доступен администраторам аккаунта) — **Работа с e-mail** — **Правила обработки для проектов** могут быть настроены правила обработки входящих сообщений, приходящих на адреса проектов: 


  * Каждое письмо, приходящее на адрес проекта, последовательно проверяется на соответствие условиям срабатывания каждого из настроенных правил.


  * В случае выполнения условий правила, проводятся описанные в правиле операции, влияющие на создаваемую по этому сообщению задачу.


  * Возможности правил аналогичны [ правилам создания задачи](Правила_создания_задач_по_e-mail.md "Правила создания задач по e-mail") по письмам, приходящим на [ внутренние адреса сотрудников](Как_поставить_задачу_по_e-mail.md "Как поставить задачу по e-mail").
