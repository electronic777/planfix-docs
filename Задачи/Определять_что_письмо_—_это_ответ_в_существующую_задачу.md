В ПланФиксе существует возможность определять, что письмо - это ответ в существующую задачу. Эта настройка находится в разделе Управление аккаунтом — Работа с e-mail — [Настройка входящих писем](Коммуникации/Настройка_входящей_почты.md). 


В ПланФиксе существует 3 способа определения письма в качестве ответа в существующую задачу: 


  * **По техническим заголовкам письма**


    Этот вариант позволяет по техническим заголовкам письма определить, что оно является ответом в существующую задачу. Это стандартная для ПланФикса схема определения связанности писем, поэтому эта настройка активирована для всех аккаунтов.


  * **По теме и отправителю**


    Этот вариант работает только для писем, тема которых начинается с Re или Fwd (то есть, из нее очевидно, что это ответ): 


  * Система находит последнюю по номеру задачу с таким же названием, как в теме письма (без учета Re и Fwd).

  * Отправитель письма должен иметь доступ к этой задаче.


    Для всех аккаунтов, зарегистрированных начиная с 20.07.2020, эта настройка по умолчанию включена.


  * **Только по отправителю**


    Этот вариант работает только для писем, отправленных контактами. Система находит последнюю по номеру задачу (в любом статусе), доступную этому контакту, и прикрепляет письмо комментарием в эту задачу.


    В простом случае это значит, что если письма приходят от контакта, то по первому письму создается задача, в которой он является постановщиком, а все остальные письма становятся комментариями в этой задаче.


## Важно


  * Когда включены все три пункта, определение письма в качестве ответа в существующую задачу происходит последовательно: сначала задачи находятся по техническим заголовкам письма, если соответствий не найдено — начинается поиск задач по теме и отправителю. Если опять ничего не найдено — поиск ведется только по отправителю письма, который указан в задаче в качестве постановщика, исполнителя или участника.
