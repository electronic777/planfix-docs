В ПланФиксе можно создавать пользовательские (то есть, добавляемые самостоятельно) [роли проекта](Роли_в_проекте.md "Роли в проекте"). Работает этот механизм следующим образом: 


  * Один из [типов пользовательских полей](Типы_пользовательских_полей.md "Типы пользовательских полей"), которые можно добавить в карточку проекта – это тип [Сотрудник](Сотрудник.md "Сотрудник").

  * Каждое добавленное в [шаблон проекта](Шаблоны_проектов.md "Шаблоны проектов") поле такого типа становится доступным в [шаблонах задач](Шаблоны_задач.md "Шаблоны задач") в списке [исполнителей](Исполнители_задачи.md "Исполнители задачи"), [участников](Участники_задачи.md "Участники задачи") и [аудиторов](Аудитор_задачи.md "Аудитор задачи").


Это позволяет придумать любую роль (или несколько), добавить их в шаблон проекта, а затем при создании по этому шаблону нового проекта выбрать конкретных сотрудников, которые будут в нем выполнять эти роли. 


Задачи проекта, созданного по такому шаблону, автоматически унаследуют эти установки и расставятся на выбранных для данного конкретного проекта сотрудников. 


## Создание пользовательской роли


  * В карточке редактирования или создания [шаблона проекта](Шаблоны_проектов.md "Шаблоны проектов") добавляем новое поле, нажав плюс.

  * Выбираем [тип поля](Типы_пользовательских_полей.md "Типы пользовательских полей") **Сотрудник** и указываем название, здесь же настраиваем [права доступа](Доступ_к_пользовательским_полям.md "Доступ к пользовательским полям") к полю:


![piTY7B.png](images/piTY7B.png)


  


  * Переносим созданное поле на нужное место в карточке проекта:


![DcS0A1.gif](images/DcS0A1.gif)


  


  * Теперь пользовательское поле будет присутствовать во всех проектах, созданных по данному шаблону.


## Использование пользовательской роли в шаблонах задач проекта


В [шаблонах задач](Шаблоны_задач_в_рамках_шаблонов_проектов.md "Шаблоны задач в рамках шаблонов проектов") внутри [шаблона данного проекта](Шаблоны_проектов.md "Шаблоны проектов") в качестве исполнителя, участника или аудитора нужно выбрать не конкретного человека, а соответствующее поле проекта: 


![LKaydD.png](images/LKaydD.png)


## Создание проекта по шаблону с пользовательскими ролями


  * Таким образом, при создании проекта по шаблону, будет достаточно один раз указать, кто в нем будет присутствовать в той или иной роли:


![MO3LQL.png](images/MO3LQL.png)


  


  * И задачи проекта будут сразу поставлены на выбранных сотрудников.


## Изменение пользователя для пользовательской роли в созданном проекте


  * Вы можете заменить сотрудника в выбранной пользовательской роли, и распространить это изменение на все задачи проекта, созданные по шаблону с использованием соответствующей роли.

  * Для этого в **Деталях проекта** укажите нового сотрудника. Система предложит применить изменение ко всем активным задачам, в шаблоне которых используется эта роль.
