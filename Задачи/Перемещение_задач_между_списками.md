Перемещение задач в [ планировщике](Прочее/Планировщик.md) приводит к изменению их параметров. Общий принцип этого изменения можно сформулировать так: 


**После перемещения в другой список (столбец) задача приобретает параметры, по которым отбираются задачи в этот список**. 


Все параметры отбора, перечисленные в условиях отбора списка Планировщика, применяются к задаче последовательно, в порядке, в котором они указаны в [списке параметров отбора](Настройка/Параметры_отбора_объектов.md). 


Если в списке условий есть несколько типовых, например, статус 1 ИЛИ статус 2, то применится только одно, последнее условие (статус 2 в данном примере). 


Такая логика используется для того, чтобы при настройке была возможность в явном виде указывать, какое условие должно примениться — для этого достаточно разместить его последним. 


## Перемещение задачи в другой столбец календаря (внутри одного списка)


Перемещение задачи в календаре задач приводит к изменению ее планируемой даты завершения:

Обратите внимание, вместе с переводом дедлайна на актуальную дату, задача перестаёт быть просроченной (больше не выделена красным цветом). 


## Перемещение задачи в другой список планировщика


Перетащим задачу из списка, в котором отобраны задачи со статусом "В работе" (условие отбора Статус задачи = В работе) в список "Выполненные" (условие отбора Статус задачи = Выполненные): 


![Q7wNZL.gif](images/Q7wNZL.gif)


  

Задача не просто переместилась из столбца в столбец — это перемещение изменило ее статус на "Выполненная". 


Еще один пример: в столбцах отобраны задачи по этапам, при этом исполнители на каждом этапе разные. 


  * Параметры отбора в первом списке: Статус = "В работе" И Исполнитель = "Виктория"

  * Параметры отбора во втором списке: Статус = "Выполненная" И Исполнитель = "Виталий"


При перемещении задачи между списками, в ней меняется не только статус — с "В работе" на "Выполненная", но и исполнитель — с Виктории на Виталия. 


![0Bw3bb.gif](images/0Bw3bb.gif)


  

Если перетаскивать задачу с зажатой клавишей "Ctrl", она попадёт в список "Выполненная", а исполнителями станут два сотрудника — Виктория и Виталий: 


![ppJxbO.gif](images/ppJxbO.gif)


  

Так происходит потому, что при перемещении задачи с зажатой клавишей **Ctrl** , система пытается дополнить некоторые поля задачи данными, указанными в условиях отбора списка-приемника, в который перемещается задача. 


Для полей, которые могут содержать только одно значение — например, системного поля "Статус" — это невозможно, поэтому данные из условий отбора в список-приемник заменяют собой данные, которые ранее хранились в этой задаче, и статус меняется с "В работе" на "Выполненная". 


А вот поле "Исполнители" поддерживает несколько значений, поэтому к Виктории добавляется и Виталий. 


На текущий момент такой механизм дополнения распространяется на 2 типа полей: 


  * Системное поле [Исполнители](Задачи/Исполнители_задачи.md).

  * Пользовательские поля типа [Набор значений](Прочее/Набор_значений.md).


## Перемещение контактов в Планировщике


  * Если перетащить контакт из списка контактов, отображаемого в Планировщике, на находящийся тут же список задач, откроется окно с предложением создать задачу. При этом контакт в создаваемой задаче будет автоматически предложен в качестве [контрагента](Прочее/Контрагент.md).

  * Если перетащить контакт на [календарь](Планирование/Календарь_по_дням.md), ПланФикс предложит создать задачу по этому контрагенту на определенное время.

  * Параметры отбора в список Планировщика влияют также на параметры создаваемых в нем [быстрым](Задачи/Быстрое_создание_задачи_в_списке.md) или [расширенным](Создание_задачи/контакта_в_списке.md "Создание задачи/контакта в списке") образом контактов.
