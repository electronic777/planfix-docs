Токены авторизации могут быть ограничены заданными уровнями доступа - scope. Поддерживаются следующие уровни доступа: 


  * **Задачи**

    * task_readonly — доступно только получение списка задач или задачи по номеру.

    * task_add — доступно только создание новых задач.

    * task_update — доступно только изменение задач.


  * **Сотрудники**

    * user_readonly — доступно только получение списка сотрудников или сотрудника по номеру.

    * user_add — доступно только создание новых сотрудников.

    * user_update — доступно только изменение сотрудников.


  * **Контакты**

    * contact_readonly — доступно только получение списка контактов или контакта по номеру.

    * contact_add — доступно только создание новых контактов.

    * contact_update — доступно только изменение контактов.


  * **Комментарии**

    * comment_readonly — доступно только получение списка комментариев или комментарий по номеру.

    * comment_add — доступно только создание новых комментариев.

    * comment_update — доступно только изменение комментариев.

    * comment_delete — доступно только удаление комментариев.


  * **Аналитики**

    * datatag_readonly — доступно только получение списка аналитик, записей аналитик или аналитики по номеру.

    * datatag_add — доступно только добавление новых записей аналитик.

    * datatag_update — доступно только изменение записей аналитик.

    * datatag_delete — доступно только удаление записей аналитик.


  * **Файлы**

    * file_readonly — доступно только получение метаданных файла или скачивания файла по номеру.

    * file_add — доступна только загрузка новых файлов.

    * file_delete— доступно только удаление файлов.
