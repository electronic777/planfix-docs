Установка конфигурации [Согласование отпуска](Согласование_отпуска.md) происходит в [ соответствующем разделе](../Настройка/Инструкция_по_установке_конфигурации.md). В процессе установки вы можете сразу настроить цепочку согласования и определить ответственных на каждом этапе. 


**1\. Название процесса согласования.** Имеет смысл дать процессу развернутое, понятное коллегам название, т.к. точно также будут называться процесс и шаблон задачи для согласования. 


**2\. Название этапов согласования.** Название каждого этапа должно передавать его смысл, например "Согласование руководителем" или "Согласование Генеральным директором" 


**3\. Кто согласовывает.** Выберите сотрудника или группу сотрудников, которые полномочны согласовывать отпуск (отгул/выходной/командировку и т.д.) и на этом этапе. 


**4\. Добавить этап.** Вы можете добавить в цепочку согласование любое необходимое количество этапов и настроить для них названия и уполномоченных сотрудников. 


**5\. Сохранить.** После того, как все этапы настроены, нажмите Сохранить для того, чтобы запустить процесс установки конфигурации. В ходе установки система автоматически сформирует нужный процесс и шаблон задачи согласования отпуска. 


После установки, внести изменения в установленную конфигурацию можно в режиме [ настройки](Настройка_конфигурации_Согласование_отпуска.md). Также вы можете [ удалить конфигурацию](../Настройка/Инструкция_по_удалению_конфигурации.md) и установить ее заново. 


## Важно


По умолчанию, шаблон задачи для согласования отпуска доступен всем сотрудникам аккаунта. Сразу после установки вы можете перейти в этот шаблон и [ настроить его доступность](../Задачи/Доступ_к_шаблонам_задач.md) только для нужных сотрудников и/или групп.
