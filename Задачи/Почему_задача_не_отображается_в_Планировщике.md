Если задача не отображается в нужном списке [планировщика](../Общие/Планировщик.md), это может быть связано с одной из причин ниже: 


## Оглавление


  * Задача не соответствует параметрам отбора списка

  * Отсутствует доступ к задаче

  * Участник — «Я» или «Группа сотрудников»

  * Ошибка в логике фильтра

  * Группировка по полю, которого нет в Объекте

  * Нет доступа к полю, по которому указана группировка

  * Неверно указан один из параметров для задач по интеграции

  * Неверная аналитика указана в данных для отображения или к ней нет доступа

  * Техническое ограничение таблицы в 150 задач

  * Автопринятие задач

  * Отработала автоматика, и задача перестала соответствовать параметрам отбора списка

  * Задачи в простых списках планировщика доступны только постановщику


## Задача не соответствует параметрам отбора списка


Задача не отображается в Планировщике, если не соответствует хотя бы одному из условий в параметрах отбора. Решить проблему можно двумя способами: 


  * Изменить параметры отбора, чтобы задача по ним проходила.

  * Изменить задачу, чтобы она подходила под параметры отбора.


Как решить проблему — смотрите в видео: 


## Отсутствует доступ к задаче


Для каждой задачи в ПланФиксе можно настроить права доступа. Сотрудники или контакты могут видеть только те задачи, в которых находятся хотя бы в одной из системных ролей: Постановщик, Исполнитель, Аудитор, Участник. 


Если у сотрудника нет доступа к задаче, он не увидит ее в Планировщике. Добавьте сотрудника хотя бы в одну из системных ролей задачи. 


Как решить проблему — смотрите в видео: 


## Участник = Я или Группа сотрудников


Задача не отобразится в списке с условием «Участник» = «Я» или «Группа сотрудников», если вы находитесь в задаче в другой роли или не состоите в указанной группе сотрудников. 


Измените параметры отбора списка или измените задачу, чтобы она отобразилась в Планировщике. 


Как решить проблему — смотрите в видео: 


## Ошибка в логике фильтра


В параметрах отбора каждой задачи есть блок «логика фильтра». Он отвечает за то, как именно фильтр будет отбирать задачи по параметрам. Например, список должен отображать задачи, где «я» — исполнитель ИЛИ участник. Если в логике есть ошибка, то фильтр может выбирать не те задачи или вообще не найти подходящие задачи под условия отбора. 


Как решить проблему — смотрите в видео: 


## Группировка по полю, которого нет в Объекте


Если указать группировку по полю, которого нет в отбираемых задачах — список будет пустым. 


Решить проблему можно двумя способами: 


  * Выбрать другое поле для группировки.

  * Добавить поле, по которому настроена группировка, в Объект.


Как решить проблему — смотрите в видео: 


## Нет доступа к полю, по которому указана группировка


Если у сотрудника нет доступа к нужному полю, список задач в Планировщике будет отображаться пустым. 


Решить проблему можно по-разному: 


  * Предоставить сотрудникам доступ к полю.

  * Выбрать другое поле для группировки.

  * Настроить список без группировки.


Как решить проблему — смотрите в видео: 


## Неверно указан один из параметров для задач по интеграции


Учитывайте параметры, которые указываете в настройках интеграции. Основные причины того, что задачи не отображаются в нужном списке — это ошибки в шаблоне или в карточке объекта, который используется для их создания. 


Решить проблему можно несколькими способами: 


  * Выбрать другой шаблон или объект для создания задач по интеграции.

  * Изменить выбранный для создания задач шаблон или объект так, чтобы он соответствовал параметрам отбора.

  * Изменить условия в параметрах отбора.


Как решить проблему — смотрите в видео: 


## В данных для отображения указана неверная аналитика или к ней нет доступа


При работе с Календарями, одна из самых частых ошибок — это неверно указанная Аналитика в настройках списка или в задаче. Если аналитика в задаче, не соответствует аналитике в данных для отображения в Календаре, то задача не отобразится в списке. 


Решить проблему можно несколькими способами: 


  * Измените прикрепленную к задаче аналитику

  * Замените аналитику в данных для отображения в Календаре.


Как решить проблему — смотрите в видео: 


## Техническое ограничение таблицы в 150 задач


При работе с Таблицей в Планировщике важно учитывать технические ограничения для отображения задач. Если в Таблице больше 150 задач, некоторые из них могут не отображаться, несмотря на то, что полностью соответствуют параметрам отбора списка. 


Для того, чтобы увидеть все задачи: 


  * Сделайте фильтр отбора более целевым, чтобы сократить количество отображаемых задач.

  * Просматривайте задачи в расширенном списке.


Как решить проблему — смотрите в видео: 


## Автопринятие задач


При постановке новых задач самому себе важно учитывать настройку автопринятия задач. Если функция активна, то при постановке задачи она будет автоматически меняет статус задачи с «Новая» на «В работе». 


Отключить автопринятие можно в настройках Объекта — в наборе статусов. 


Как решить проблему — смотрите в видео: 


## Отработала автоматика и задача перестала соответствовать параметрам отбора


Если автоматический сценарий или кнопка изменит задачу так, что она перестанет соответствовать параметрам отбора списка — она перестанет в нем отображаться. 


Чтобы избежать такой проблемы: 


  * Измените параметры отбора так, чтобы задачи отображались и после срабатывания автоматики.

  * Изначально настраивайте отбор задач в список Планировщика с учетом работы автоматики.


Как решить проблему — смотрите в видео: 


## Задачи в простых списках планировщика доступны только постановщику


Простой список задач — это список Планировщика без параметров отбора. Он часто служит для быстрой постановки задач самому себе. Если вы работаете с таким списком в команде, помните, что к каждой задаче в таком списке надо предоставлять доступ для коллег. По умолчанию задачи в простых списках доступны только постановщику. 


Для того, чтобы коллеги видели задачи, добавьте их в одну из системных ролей задачи: Постановщик, Исполнитель, Аудитор, Участник.
